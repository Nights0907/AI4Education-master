# AI4Education

AI4Education（沐斯 AI 数字人教师）是一个面向中小学学习场景的 AI 个性化教育后端项目。项目以学生错题和学习过程数据为核心，结合 RAG 检索增强、大语言模型、多模态题目录入和学生画像能力，为学生提供个性化解题、启发式问答和学情分析服务。

## 项目背景

在传统学习过程中，学生经常会遇到以下问题：

- 错题整理成本高：拍照、分类、标注知识点和回顾都依赖人工完成，效率较低。
- 解题辅导不够个性化：普通搜题或问答工具往往只给出标准答案，无法结合学生已有薄弱点进行讲解。
- 学情数据难以沉淀：学生的错题、答题记录和提问过程分散在不同场景中，难以形成持续更新的学习档案。
- 大模型回答存在不稳定性：直接调用大模型容易出现知识不匹配、解释不贴合题目、缺少教学引导等问题。
- 交互等待时间长：大模型完整生成答案耗时较久，前端等待体验较差。

本项目希望通过“错题数据沉淀 + 学生画像 + RAG 增强 + 大模型教学对话”的方式，让 AI 不只是给答案，而是能够像教师一样结合学生情况进行引导、讲解和追问。

## 解决什么痛点

AI4Education 主要解决个性化学习场景中的三个核心痛点：

1. **从“答案工具”到“教学助手”**
   
   项目通过 Prompt Engineering 设计启发式问答、个性化解析和费曼学习法等对话流程，让 AI 在回答时更关注学生的思考过程，而不是只输出最终答案。

2. **从“零散错题”到“动态学生画像”**
   
   系统会基于学生的错题、答题记录和交互数据，归纳知识点薄弱环节，维护可持续更新的学生学习档案，为后续个性化讲解提供依据。

3. **从“直接问大模型”到“可信知识增强”**
   
   项目基于 Milvus 构建数学题库向量知识库，并结合 BM25、向量检索和重排序模型，为大模型提供更相关的参考资料，降低回答偏离题目或缺少依据的问题。

## 主要功能

### 1. 学生账号与学习档案

- 支持学生注册、登录、登出和基础资料维护。
- 记录学生答题、错题和 AI 对话行为。
- 根据学习行为维护学生个性化画像，包括知识点掌握情况、错题分布和学习历史。

### 2. 错题与题目管理

- 支持题目图片上传和文件管理。
- 支持错题基础信息、具体题目、标记、笔记、位置和历史记录管理。
- 支持对错题进行知识点归类、标签化管理和后续复习追踪。

### 3. AI 解题与错因分析

- 调用 OCR 和大模型识别题目内容。
- 分析学生错误答案，生成错因解释、知识点提示和学习建议。
- 结合学生画像生成更贴合个人情况的讲解内容。

### 4. AI 个性化教学对话

- 启发式问答：通过追问和提示引导学生主动思考。
- 个性化解析：结合学生薄弱点调整讲解方式和详细程度。
- 费曼学习法：让学生尝试复述和讲解，AI 根据表达情况判断掌握程度。
- 多轮对话记忆：使用 MongoDB 持久化教学对话历史，保证上下文连续。

### 5. RAG 检索增强

- 基于 Milvus 构建数学题库向量知识库。
- 支持文档向量化、上传、重建和状态查询。
- 使用混合检索和重排序技术，为大模型提供更相关的参考题目和解题资料。

### 6. 流式响应体验

- 使用 SSE（Server-Sent Events）实现 AI 回答流式返回。
- 将“等待完整答案生成”优化为“边生成边展示”，降低用户感知等待时间。

## 主要技术点

### 后端架构

- 使用 Spring Boot 构建后端 API 服务。
- 使用 MyBatis-Plus 简化数据库访问。
- 使用 Sa-Token 处理登录认证与会话管理。
- 使用 Redis / Spring Cache 支撑缓存能力。

### 多源数据存储

- MySQL：存储学生基础信息、错题记录、历史记录等结构化数据。
- MongoDB：存储 AI 教学场景下的多轮对话记忆。
- Milvus：存储题目和文档的向量化结果，支撑 RAG 检索。
- 阿里云 OSS：存储学生上传的题目图片、音频等非结构化文件。

### RAG 检索增强

- 基于 5000+ 道数学题目构建向量化题库。
- 采用“题干 / 问题 - 核心知识点 - 解题思路”的混合分块策略，提高题目语义表达质量。
- 融合 BM25 关键词检索与向量相似度检索，兼顾关键词精准匹配和语义相似召回。
- 引入 Cohere Rerank 对召回结果进行二次精排，将回答语义相关度提升约 15%。
- 开发 RAG 评估模块，通过召回率、相关性和回答质量等指标支持检索策略迭代。

### 大模型与 Prompt Engineering

- 支持 DeepSeek、通义千问等大模型能力接入。
- 将提示词模板集中管理在 `src/main/resources/prompts/`，避免在业务代码中硬编码长 Prompt。
- 通过 Prompt 模板约束模型输出风格，使回答更符合启发式教学、个性化教学和费曼学习法等教育场景。

### 性能与体验优化

- 使用 SSE 实现大模型回答的流式传输。
- 将前端感知响应时间从约 60 秒优化至约 0.6 秒。
- 使用缓存和分层存储降低重复计算与外部服务调用成本。

## 如何运行项目

### 1. 环境要求

请先准备以下环境：

- JDK 17+
- Maven 3.6+
- MySQL 8.x
- MongoDB 4.x+
- Milvus 2.x（使用 RAG 能力时需要）

### 2. 初始化数据库

创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS education DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
```

导入初始化 SQL：

```bash
mysql -h127.0.0.1 -uroot -p education < src/main/resources/mysql/education.sql
```

### 3. 配置环境变量

项目默认配置位于 `src/main/resources/application.properties`。

### 4. 启动依赖服务

运行项目之前，需要确保以下服务已经启动：

- MySQL
- MongoDB
- Milvus（使用 RAG 检索能力时需要）

### 5. 启动后端服务

使用本机 Maven 启动项目：

```bash
mvn spring-boot:run
```

服务默认监听：

```text
http://localhost:8443/api
```

### 6. 构建与测试

```bash
mvn clean test
mvn clean package
```

构建产物位于 `target/` 目录。
